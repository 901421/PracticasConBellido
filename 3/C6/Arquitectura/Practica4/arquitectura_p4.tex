\section{Introducción}
El presente documento detalla la arquitectura del sistema de gestión de pedidos e inventario desarrollado para la Práctica 4. El sistema evoluciona desde una solución monolítica hacia un ecosistema de microservicios orquestados mediante \textbf{Kubernetes}. Se integran patrones de comunicación síncrona (REST) y asíncrona (MOM), seguridad perimetral y persistencia descentralizada (políglota).

\section{Conclusión y Patrones Aplicados}
\begin{itemize}
    \item \textbf{API Gateway:} Punto único de entrada con seguridad por token.
    \item \textbf{Descentralización de Datos:} Persistencia independiente para Pedidos y Stock.
    \item \textbf{Resiliencia:} Uso de \textit{PersistentVolumeClaims} para durabilidad de estado.
    \item \textbf{Sincronía vs Asincronía:} Orquestación híbrida para optimizar la experiencia de usuario.
\end{itemize}
